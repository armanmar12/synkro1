# Synkro: Бизнес-контекст и процессы

Обновлено: 2026-02-20
Статус: живой документ (обновляется при изменениях продукта)

## 1. Что это за сервис
Synkro - B2B-сервис для автоматизации аналитики клиентских диалогов и отчетности.
Центр системы - веб-кабинет, где клиент подключает интеграции (Supabase, amoCRM, Radist, AI, Telegram), проверяет их и управляет процессом отчетов.

Ключевое ограничение:
- amoCRM используется только в режиме read-only (чтение данных, без записи обратно в amoCRM).

## 2. Бизнес-задача
Сократить ручную работу по сбору/сведению данных из разных источников и ускорить получение управленческих выводов по коммуникациям с клиентами.

Проблема клиента:
- данные о сделках и диалогах разрознены;
- сложно быстро получить сводный отчет;
- менеджеры и руководители тратят время на ручной анализ.

Ценность Synkro:
- единая точка настройки интеграций;
- прозрачная проверка подключений;
- консолидированная подготовка данных для отчетов;
- основа для регулярной автоматизированной аналитики.

## 3. Основные бизнес-сценарии
## 3.1 Онбординг клиента
- Создается tenant (компания/клиент в системе).
- Назначаются роли пользователей.
- Клиент или админ настраивает интеграции в Settings.
- Проверяются подключения к внешним сервисам.

## 3.2 Подготовка данных
- Из amoCRM читаются сделки/контакты и связанная метаинформация.
- Из Radist читаются диалоги и сообщения.
- Данные приводятся к нормализованному виду и пишутся в клиентский слой хранения (Supabase).

## 3.3 Аналитика и отчет
- Формируется пакет данных за период.
- AI анализирует диалоги и формирует текстовые выводы.
- Отчет сохраняется и может быть отправлен в Telegram.

## 3.4 Операционное использование
- Пользователь заходит в Dashboard, смотрит отчеты и статусы.
- При необходимости запускает отчет вручную (целевой режим - по кнопке/по расписанию).
- При сбоях смотрит статус и ошибку по шагам.

## 4. Модель монетизации (рабочие варианты)
## 4.1 Подписка по тарифам (SaaS)
- Тариф по количеству tenant/пользователей/объему обрабатываемых диалогов.
- Базовый тариф: ручной запуск и ограниченный объем.
- Pro/Enterprise: автоматизация по расписанию, расширенная аналитика, SLA.

## 4.2 Onboarding + интеграционный пакет
- Разовая оплата за первичную настройку и подключение всех интеграций.
- Подходит для клиентов, которым нужна быстрая постановка "под ключ".

## 4.3 Managed service
- Отдельный тариф, где команда Synkro сопровождает качество данных и регулярные отчеты.
- Выше чек, но меньше операционной нагрузки на клиента.

## 5. Границы ответственности сервиса
Synkro:
- хранит конфиги интеграций в центральной БД;
- управляет ролями и доступом;
- запускает/координирует шаги интеграций и отчетов.

Внешние системы:
- amoCRM, Radist, AI-провайдер, Telegram и клиентский Supabase остаются источниками/приемниками данных по API.

## 6. Текущее состояние продукта
Реализовано:
- каркас кабинета (login, dashboard, reports, profile, settings);
- модели данных для tenant/ролей/интеграций/запусков/отчетов/аудита;
- формы настройки интеграций и проверки доступности.

Частично/в плане:
- полный production-пайплайн "сбор -> анализ -> рассылка";
- детальная оркестрация шагов через Celery с наблюдаемым прогрессом в UI.
