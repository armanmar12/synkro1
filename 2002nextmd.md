Настройки, которые ты описал, это уже рабочая бизнес-логика: формы, сохранение в БД, шифрование ключей, проверки интеграций, фоновые задачи и запись в Supabase. Этого кода пока нет, поэтому “настроить” нечего.

Чтобы это появилось, нужно сделать следующий слой (по шагам):

1. Auth + роли (RBAC)
`/login` должен реально логинить, роли `super_admin/admin_lite/user` должны скрывать/открывать “Настройки”.

2. Модели в нашей БД (центральной)
`Tenant`, `IntegrationConfig` (Supabase/amo/Radist/Gemini/Telegram), `JobRun`, `AuditLog`.
Ключи хранить шифрованно и после сохранения показывать `***`.

3. Settings Wizard (реальный, не макет)
Для каждого шага: поля → “Проверить” → “Тест (3 диалога)” → “Записать в Supabase” → статус.

4. Интеграции (серверные модули)
- amoCRM: читать leads + links + phones (используем опыт из `amo testapi`).
- Radist: читать chats/messages (используем опыт из `test api`).
- Supabase: применить SQL/проверить доступ/делать upsert.
- Gemini: проверка ключа + анализ пакета.
- Telegram: тест-сообщение + отправка отчета.

5. Кнопка “Принудительный отчет” и очередь задач
Celery пайплайн: amo → Supabase → Radist → Supabase → AI → Supabase → Telegram, с прогрессом для UI.
